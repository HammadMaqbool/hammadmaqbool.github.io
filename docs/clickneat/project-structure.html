Project Structure

The project is developed with .NET 8 using ASP.NET MVCCore Technology and MSSqlServer as a database provider.
For the real time notifications the project is using SignalR Service.

Project Files - Folders

We have the following file structure

  • Root Folder:

    Contains the solution file (.sln) and project files (.csproj).

  • wwwwroot:

    Holds static files such as CSS, JavaScript, images, and libraries (e.g., Bootstrap, jQuery).

    • css/: Contains CSS stylesheets.
    • js/: Contains JavaScript files for client-side logic.
    • lib/: Includes external libraries like Bootstrap and jQuery.
    • adminend/: Contains complete adminend theme of the site and its files
    • frontend/: Contains complete frontend theme of the site and its files
    • microsoft/signalr/: Contains complete SignalR files
    • uploads/: containst the sub-folder which are holding the images of products, Deals and all other which we upload from the code.
  • Areas

    It contains the Admin Panel Controllers, Views and Services folder where as Services folder contains the ImageService.cs which is responsible for uploading and deleting the images files from the uploads/ sections folder.

  • Controllers

    This folder contains the Cart, Home and Order Controller which is responsible for the frontend of the site.

  • Data

    This folder contains the file AppDbContext.cs which is responsible for controlling and maintaing the EFCore DbContext related things.

  • Hubs

    This folder containst he Orderhub.cs file which is responsible to send SignalR Messages

  • Migrations

    The Auto generated folder for the EFCore and do not delete it if you havent read about it on Setup page

  • Models

    This folder contains all the model classes which we are using to generate the database and handle all the pages, It is recommended to run the migration if you make changing in any of these models to update the database - method to run the migrations is already discussed in the setup page.

  • Services

    This folder contains two Services

    • Cart/Cart.cs responsible to add, remove, increase and decrese the data into the cart along with the cartitem.cs class.
    • EMail/EmailService.cs responsible to Send the mails about the confirmation and cancilation of the order.
  • ViewComponents

    Holds the view components of the project

  • ViewModels

    Holds the view models of the project

  • Views

    Holds the views of the project responsible for the frond-end not admin panel

  • appsettings.json

    Holds the connection string which you are supposed to change and details are provided in the setup section about changing it.

  • libman.json

    holds the details of the Js libraries which we are using in the project - in our case it is focussing on the SignalR.

  • Program.cs

    Main driving class of the whole project.
